# ml_midterm_project

### EDA
- Ймовірність відкриття депозиту залежить від віку нелінійно: найнижча конверсія спостерігається у групі 35–50 років. Можлива причина — фінансові зобов’язання або менша чутливість до маркетингових пропозицій.  
- Самотні клієнти демонструють вищу ймовірність відкриття депозиту.  
- Найсильніший вплив мають ознаки, пов’язані з попередніми кампаніями: успішний результат попередньої кампанії підвищує ймовірність відкриття депозиту приблизно в 2 рази.  
- Кількість контактів корелює з ймовірністю відкриття депозиту, але може відображати стратегію банку, а не прямий причинний вплив.  
- Ознака `duration` має великий вплив на таргет, але **не використовується у фінальній моделі**, оскільки її значення недоступне до моменту дзвінка.  
- Соціально-економічні показники (Euribor, рівень зайнятості) корелюють із цільовою змінною, але відображають загальний економічний контекст, а не індивідуальну поведінку клієнтів.

### Вибір метрики
- Основна метрика: **F2-score**, бо вона надає більшу вагу recall порівняно з precision.  
- Бізнес-пріоритет: **False Negatives** більш критичні, ніж False Positives.  
- Для моделей підібрано оптимальний **threshold**, щоб максимізувати recall відповідно до бізнес-пріоритетів.

### Препроцесинг
- **Розподіл на набори:** тренувальний, валідаційний, тестовий (Stratified Split)  
- **Feature engineering:**  
  - Для kNN та Logistic Regression створені interaction features:  
    - `job_prev_interaction` — комбінація `job` та `previous`  
    - `marital_campaign_interaction` — комбінація `marital` та `campaign`  
    - `education_empvar_interaction` — комбінація `education` та `emp.var.rate`  
  - Для Decision Tree та бустингових моделей — групування віку та сезону  
- **Масштабування ознак:** стандартизація для моделей, чутливих до масштабу (kNN, Logistic Regression)  
- **Ресамплінг:** SMOTE на тренувальному наборі для kNN та Logistic Regression  
- **Примітки:**  
  - `duration` використовується тільки для аналізу, не включається у фінальні моделі  
  - Interaction та групування будуються тільки на тренувальних даних, щоб уникнути витоку інформації

### Використані моделі
- Logistic Regression  
- k-Nearest Neighbors (kNN)  
- Decision Tree  
- Gradient Boosting (Sklearn + Hyperopt для тюнінгу)
### Результати моделей
| Модель | Гіперпараметри | F2-score (Train) | F2-score (Validation) | Коментар |
|--------|----------------|-----------------|---------------------|----------|
| Logistic Regression | `C=0.1, max_iter=1000, penalty='elasticnet', solver='saga', class_weight='balanced', l1_ratio=0.5` | 0.68 | 0.74 | Хороша для базового розуміння; interaction features підвищили recall |
| k-Nearest Neighbors | `n_neighbors=11, p=2, weights='uniform'` | 0.75 | 0.67 | Чутлива до масштабування; SMOTE допоміг збалансувати класи |
| Decision Tree | `class_weight='balanced', max_depth=7, min_samples_leaf=10, min_samples_split=10` | 0.66 | 0.63 | Найкраща модель без ресамплінгу; легка для інтерпретації |
| Gradient Boosting (Hyperopt) | `colsample_bytree=0.579, gamma=0.231, learning_rate=0.094, max_depth=7, min_child_weight=7, n_estimators=225, reg_alpha=0.078, reg_lambda=0.337, subsample=0.720` | 0.83 | 0.77 | Гіперпараметри підібрати з Heperopt |
### Висновки та рекомендації
- Найсильніший вплив на ймовірність відкриття депозиту мають попередні кампанії та соціально-демографічні фактори  
- Моделі бустингу забезпечують кращу точність при високому recall  
- Для покращення моделі можна додатково проаналізувати поведінкові патерни клієнтів та динамічні часові ознаки
